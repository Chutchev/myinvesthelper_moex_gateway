# MOEX Gateway

Go-шлюз на Fiber v3 для получения рыночных данных MOEX и данных Банка России.
Сейчас это компилируемый HTTP-scaffold: health-check работает, а доменные
сервисы остаются заглушками и возвращают `501 Not Implemented`. Исходящие
запросы к MOEX и ЦБ, парсинг ответов, кэширование и бизнес-логика пока не
реализованы.

## HTTP API

| Метод | Маршрут | Текущее поведение |
|---|---|---|
| `GET` | `/health` | `200 OK`, `{"status":"ok"}` |
| `GET` | `/v1/bonds/{isin}` | Проверка ISIN, затем `501 Not Implemented` |
| `GET` | `/v1/bonds?limit=40` | Проверка `limit` (1–200), затем `501 Not Implemented` |
| `GET` | `/v1/cbr/rates` | `501 Not Implemented` |

Swagger UI доступен по адресу `/swagger/index.html`. Спецификация генерируется
из Go-аннотаций; вручную редактировать файлы в `docs/` не нужно.

## Структура

```text
cmd/gateway/          точка входа и обработка сигналов
internal/app/         жизненный цикл HTTP-сервера
internal/config/      конфигурация из переменных окружения
internal/httpserver/  маршруты, валидация и JSON-ответы
internal/moex/        контракты и заглушки домена MOEX
internal/cbr/         контракты и заглушки домена ЦБ
internal/cache/       интерфейс кэша
internal/apperrors/   общие ошибки приложения
```

Gateway отвечает только за получение, нормализацию и выдачу данных MOEX/ЦБ.
Python-приложение сохраняет ответственность за БД, оркестрацию `bond_sync` и
остальную прикладную логику.

## Запуск

Требуется Go 1.25 или новее. Go может автоматически скачать подходящий toolchain
по директиве из `go.mod`. Все настройки имеют безопасные значения по
умолчанию, перечисленные в `.env.example`. Для локального переопределения:

```sh
cp .env.example .env
set -a
. ./.env
set +a
make run
```

Приложение читает переменные процесса и не загружает `.env` автоматически.

## Команды

- `make run` — запустить gateway;
- `make test` — выполнить тесты;
- `make vet` — запустить статический анализ;
- `make build` — собрать все пакеты;
- `make fmt` — отформатировать Go-код;
- `make swagger` — обновить generated Swagger-файлы в `docs/`.
