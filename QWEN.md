# MOEX Gateway — Project Context

Go-шлюз на Fiber v3 для получения рыночных данных MOEX (облигации) и данных Банка России (ключевая ставка, прогноз). Приложение отвечает только за получение, нормализацию и выдачу данных — бизнес-логика и БД находятся в отдельном Python-сервисе.

## Архитектура

```
cmd/gateway/          Точка входа, обработка сигналов, Swagger-аннотации
internal/app/         Жизненный цикл HTTP-сервера (Fiber)
internal/config/      Загрузка конфигурации из переменных окружения
internal/httpserver/  Роутер, хендлеры, валидация, JSON-ответы
internal/moex/        Контракты и заглушки домена MOEX (облигации)
internal/cbr/         Контракты и заглушки домена ЦБ (ставки)
internal/cache/       Интерфейс кэша (пока не реализован)
internal/apperrors/   Общие ошибки приложения (ErrNotImplemented)
docs/                 Сгенерированная Swagger-спецификация
```

## HTTP API

| Метод | Маршрут | Поведение |
|---|---|---|
| `GET` | `/health` | `200 OK`, `{"status":"ok"}` |
| `GET` | `/v1/bonds/{isin}` | Проверка ISIN, затем `501 Not Implemented` |
| `GET` | `/v1/bonds?limit=40` | Проверка `limit` (1–200), затем `501 Not Implemented` |
| `GET` | `/v1/cbr/rates` | `501 Not Implemented` |
| `GET` | `/swagger/*` | Swagger UI |

Все доменные сервисы пока возвращают `501 Not Implemented`. Исходящие запросы к MOEX и ЦБ, парсинг ответов, кэширование и бизнес-логика — в планах.

## Требования

Go 1.25+. Go автоматически скачает подходящий toolchain по директиве в `go.mod`.

## Конфигурация

Все настройки через переменные окружения (см. `.env.example`). Приложение не загружает `.env` автоматически — экспортируйте вручную:

```sh
set -a; . ./.env; set +a
```

Переменные:
- `SERVER_HOST` / `SERVER_PORT` — адрес и порт (по умолчанию `0.0.0.0:8080`)
- `MOEX_BASE_URL` — базовый URL MOEX ISS API
- `CBR_KEY_RATE_URL` / `CBR_FORECAST_PAGE_URL` — URL Банка России
- `HTTP_TIMEOUT` — таймаут HTTP-клиента (по умолчанию `10s`)
- `MARKET_CACHE_TTL` — TTL кэша рыночных данных (по умолчанию `15m`)
- `LOG_LEVEL` — уровень логирования

## Тесты

Тесты расположены рядом с исходниками (`*_test.go`). Покрытие включает роутер, хендлеры, клиентские и сервисные слои MOEX/ЦБ, типы и валидацию.

## Соглашения

- **Код:** Go-стандарты, `gofmt`, пакеты `snake_case`, интерфейсы с суффиксом `Service`
- **Заглушки:** Доменные сервисы реализованы как `StubService` и возвращают `apperrors.ErrNotImplemented`
- **Swagger:** Спецификация генерируется из Go-аннотаций через `swag init`. Файлы в `docs/` не редактировать вручную
- **Ошибки:** Единая `ErrNotImplemented` для незавершённых эндпоинтов, HTTP-статус `501`
- **JSON-ответы:** Единый формат `{"error": "message"}` через `ErrorResponse`

## Планы

Реализовать исходящие HTTP-клиенты к MOEX и ЦБ, парсинг ответов, кэширование (интерфейс `cache.Cache` уже определён), и бизнес-логику в сервисах `moex` и `cbr`.
